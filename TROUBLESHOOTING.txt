TROUBLESHOOTING - crashes and common issues
===========================================

CRASH: Missing global shader FTransferFunctionOidnCS
----------------------------------------------------
Cause: NOT a weak PC / low FPS issue. This is a fatal shader error.

Our old Ultra Max preset forced hardware ray tracing:
  r.RayTracing=1
  r.Lumen.HardwareRayTracing=1

That enables OIDN (Open Image Denoise) shaders for RT denoising.
Subnautica 2 shipping build does not include that shader path, so the game
crashes on startup or when loading a level.

FIX (already applied in preset v2.1):
  r.Lumen.HardwareRayTracing=0
  r.RayTracing=0

Ultra Max still uses Lumen GI and Lumen reflections (software path).
Quality remains very high; only GPU hardware RT is disabled.

If you still crash:
  1. Close the game.
  2. Remove Read-only from Engine.ini in AppData.
  3. Delete Engine.ini OR re-copy 05_UltraMax\Engine.ini from this mod.
  4. Set Read-only again.
  5. Or apply 03_Medium or 04_High preset instead.

Quick emergency fix - add to Engine.ini [SystemSettings]:
  r.RayTracing=0
  r.Lumen.HardwareRayTracing=0


Engine.ini deleted after launch
-------------------------------
Set Engine.ini to Read-only in file Properties.


Settings reset after opening graphics menu
------------------------------------------
In-game menu overwrites GameUserSettings.ini. Re-apply preset or avoid menu.


Low FPS on Ultra Max (no crash)
-------------------------------
Use 04_High or 03_Medium. Lower r.Streaming.PoolSize if VRAM is limited.
Use 06_Potato or 01_Minimum on very weak PCs.


Preset made game SLOWER or stuttery (even Potato)
-------------------------------------------------
Valid report for mod versions before v2.2. Causes:

1. Engine.ini disabled Nanite (r.Nanite=0) on low presets.
   Subnautica 2 is built for Nanite; turning it off forces a slower
   fallback path and can tank FPS.

2. Triple resolution scaling on Potato:
   dynamic resolution + r.ScreenPercentage=70 + DLSS 40% scale.
   That causes hitching and blurry output without real FPS gain.

3. AMD GPU + DLSS preset = broken upscaling path (use 07_AMD_FSR).

4. Full file replace from Nexus without merging (loses tuned defaults).

FIX (v2.2+):
  1. Close game.
  2. restore-backup.bat OR delete Engine.ini (remove Read-only first).
  3. Re-apply preset with updated mod files.

If still worse than vanilla:
  A) Settings-only (no Engine.ini):
     powershell -File apply-preset.ps1 -Preset potato -SettingsOnly
  B) Delete Engine.ini entirely and tune only in-game menu.
  C) Merge ONLY [ScalabilityGroups] + SN2SettingsLocal from preset;
     skip Engine.ini completely.


AMD GPU
-------
Use 07_AMD_FSR or set UpscalingMethod=U_FSR in GameUserSettings.ini.
